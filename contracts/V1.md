# v1 Contract — Frozen Wire Format

This directory holds the v1 contract surface for `health-data-hub`. The
HealthSave iOS app (App Store ID 6759843047), the
[`health-data-to-mqtt`](https://github.com/bietiekay/health-data-to-mqtt)
community bridge, and any other v1 client expect every route and
payload shape listed here to remain stable across the v2.x line.

> **Rule.** A change that breaks `contracts/openapi/v1.locked.json` is a
> v1 contract change. It must be deliberate, semver-bumped, and
> coordinated with an iOS app release. CI fails the build if the live
> OpenAPI drifts from the lock without an explicit regen.

## See also

- [`IOS_CROSS_CHECK.md`](IOS_CROSS_CHECK.md) — audit comparing the
  HealthSave iOS app's wire format against the OpenAPI lock. The
  iOS-narrow subset (3 routes) is pinned separately by

## How the freeze is enforced

Three contract tests in `tests/contract/api_v1/test_v1_contract.py`:

| Test | What it asserts |
|------|-----------------|
| `test_v1_routes_present` | every route in `V1_ROUTES_FROZEN` is still served by the live FastAPI app |
| `test_no_unexpected_v1_routes` | new top-level routes don't sneak in without being added to the frozen inventory |
| `test_openapi_matches_locked_snapshot` | the live `app.openapi()` matches `contracts/openapi/v1.locked.json` byte-for-byte |

Run them with:

```bash
pytest tests/contract/api_v1/ -v
```

CI runs the full pytest suite, so contract drift fails the PR build.

## Regenerating the lock

When a v1 change is intentional (rare, requires iOS coordination):

```bash
make regen-lock      # regenerate inside Docker (pinned deps)
make check-lock      # CI-style drift check (host Python OK)
```

> **Always regenerate inside Docker.** The OpenAPI JSON depends on
> FastAPI/Pydantic versions; generating on a host with different
> versions than the pinned ones (`pyproject.toml` dependencies)
> produces serializer-noise drift that fails CI even when the wire
> contract is unchanged. The Makefile target builds the project's
> Docker image and regenerates from there.

The diff in `contracts/openapi/v1.locked.json` is the v1 wire-format
change. Commit it with a message that names the iOS coordination plan.

## The v1 inventory

Twelve routes. The full payload shapes are in `openapi/v1.locked.json`;
this table is the index.

### Health & readiness

| Method | Path | Purpose | Auth |
|--------|------|---------|------|
| GET | `/health` | Process liveness — always 200 OK | none |
| GET | `/api/health` | App-friendly alias of `/health` | none |
| GET | `/ready` | DB connectivity check; 503 on failure | none |

### Apple Health ingest (HealthSave iOS contract)

| Method | Path | Purpose | Auth |
|--------|------|---------|------|
| POST | `/api/apple/batch` | Receive a HealthKit metric batch | `x-api-key` if `API_KEY` set |
| GET | `/api/apple/status` | Per-table record counts (flat, top-level metric keys — iOS parses without wrapper) | `x-api-key` if `API_KEY` set |

### Insights (analysis surface)

| Method | Path | Purpose | Auth |
|--------|------|---------|------|
| GET | `/api/insights/latest` | Most recent daily briefing + weekly summary | `x-api-key` if set |
| GET | `/api/insights/daily` | Daily briefing narrative | `x-api-key` if set |
| GET | `/api/insights/weekly` | Weekly summary narrative | `x-api-key` if set |
| GET | `/api/insights/anomalies` | Persisted anomaly findings (filterable by `since`, `severity`) | `x-api-key` if set |
| GET | `/api/insights/trends` | Persisted trend findings (filterable by `period`) | `x-api-key` if set |
| POST | `/api/insights/trigger` | Run a supported analysis job inline | `x-api-key` if set |

### Observability

| Method | Path | Purpose | Auth |
|--------|------|---------|------|
| GET | `/metrics` | Prometheus exposition (counters + histograms) | none |

### Auto-generated (FastAPI built-ins, NOT v1 contract)

`/docs`, `/docs/oauth2-redirect`, `/redoc`, `/openapi.json` — useful for
exploration, not part of the freeze.

## Hard rules across v2.x

1. **Don't break the iOS app.** Every payload shape served on the
   routes above is contracted. `/api/apple/status` specifically must
   stay flat with top-level metric keys (no `{"status": "ok",
   "counts": {...}}` wrapper). See the inline comment in
   `server/api/status.py`.
2. **Don't add a route without adding it to `V1_ROUTES_FROZEN` and
   regenerating the lock.** New routes are part of the v2 surface;
   they live in `/v2/*` paths and are contracted separately.
3. **Don't change auth semantics on a v1 route.** `x-api-key`
   enforcement on the routes that already require it is part of the
   contract.
4. **Don't remove a route to "clean up."** Removal is deprecation;
   deprecation is a coordinated iOS-app release plus a deprecation
   header for a window, not a delete.

## What is NOT in v1

- WebSockets
- SSE streaming routes
- Plugin-emitted custom endpoints
- Anything under `/v2/*`
- Anything under `/agents/*`

These will land in v2 with their own contract surface, separately
versioned.

## Where the v2 plan lives

The full v2 architecture plan (apps / packages / plugins / contracts /
deploy split, plugin SDK, agent runtime, web dashboard) is the
project's next-generation scaffolding direction. v1 is preserved
underneath every v2 change via this contract surface.
